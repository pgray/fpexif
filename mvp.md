## Verification

### Test Data Directories

There are two test data directories for verification:

| Directory | Flag | Description | Use Case |
|-----------|------|-------------|----------|
| `/fpexif/raws/` | (default) | Small curated set of RAW files | Quick iteration and development |
| `/fpexiv/data.lfs/` | `--data-lfs` | Large dataset with more camera models | Comprehensive verification before release |

### Testing Workflow

1. **During Development** - Test against default raws for quick feedback:
```bash
./bin/mfr-test canon --verbose
./bin/mfr-test nikon --verbose
./bin/mfr-test sony --verbose
./bin/mfr-test fujifilm --verbose
./bin/mfr-test panasonic --verbose
./bin/mfr-test olympus --verbose
```

2. **Before Marking Complete** - Test against LFS dataset to catch edge cases:
```bash
./bin/mfr-test canon --data-lfs --verbose
./bin/mfr-test nikon --data-lfs --verbose
./bin/mfr-test sony --data-lfs --verbose
./bin/mfr-test fujifilm --data-lfs --verbose
./bin/mfr-test panasonic --data-lfs --verbose
./bin/mfr-test olympus --data-lfs --verbose
./bin/mfr-test pentax --data-lfs --verbose
./bin/mfr-test samsung --data-lfs --verbose
./bin/mfr-test sigma --data-lfs --verbose
./bin/mfr-test minolta --data-lfs --verbose
./bin/mfr-test kodak --data-lfs --verbose
```

### Regression Prevention

Use baselines to track progress and prevent regressions:

```bash
# Save baseline before starting work on a manufacturer
./bin/mfr-test canon --save-baseline

# After making changes, check for regressions
./bin/mfr-test canon --check

# Test against larger dataset
./bin/mfr-test canon --data-lfs --check
```

**IMPORTANT**: Always test against BOTH directories before marking tags as complete. The LFS dataset often reveals edge cases and additional camera models not present in the smaller raws set.
